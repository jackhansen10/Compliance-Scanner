# Contributing

Thanks for your interest in contributing! This project focuses on SOC 2
evidence collection and reporting for AWS. Please keep changes scoped and
aligned with that goal.

## Before you start

- For substantial changes, open an issue to discuss the approach first.
- Check existing issues and PRs to avoid duplication.

## Development setup

1. Create and activate a virtual environment.
2. Install dependencies from `requirements.txt`.
3. Run the test suite.

See `README.md` for the exact commands.

## Making changes

- Keep PRs focused on a single goal.
- Include tests when you change behavior or fix a bug.
- Update documentation and CLI help when you add or change flags.
- Avoid introducing new dependencies unless needed.

## Submitting a PR

- Use a descriptive title and summary.
- Link the related issue if applicable.
- Ensure tests pass locally.

## Code of Conduct

By participating, you agree to abide by the `CODE_OF_CONDUCT.md`.
